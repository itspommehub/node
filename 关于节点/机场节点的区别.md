## 🌍 一、线路类型对比

决定“速度”和“稳定性”的关键！

|   |   |   |   |   |
|---|---|---|---|---|
|类型|🌐 全称|💬 说明|⚖️ 优缺点|🎯 适用场景|
|🌀 **中转线路**|Relay / Proxy Transit|数据先到国内服务器再出国|✅ 成本低  <br>⚠️ 稳定性差|普通上网、轻度使用|
|🌏 **BGP 动态线路**|Border Gateway Protocol|动态路由，自动选最优出口|✅ 智能路由、延迟低  <br>❌ 成本略高|通用日常使用|
|⚡ **IPLC 专线**|International Private Leased Circuit|点对点物理国际专线|✅ 极稳无丢包  <br>❌ 昂贵|高端套餐、企业远程|
|🛠️ **IEPL 专线**|International Ethernet Private Line|企业级以太网专线|✅ 稳定、低延迟  <br>❌ 成本高|视频会议、游戏、办公|
|🧭 **BGP + IEPL 混合**|Hybrid Line|动态与专线结合|✅ 稳定+灵活  <br>❌ 依机场质量|高端用户、稳定党|
|🌐 **Anycast**|Global Routing|全球多节点共享IP|✅ 可用性高  <br>❌ 速度波动大|多地区访问、临时节点|
|🧩 **原生线路**|Native IP Route|真实地区IP（非复用）|✅ 可解锁流媒体  <br>❌ 成本高|Netflix、ChatGPT等|
|🧱 **CN2 / CMI 优化**|China Telecom CN2 / China Mobile Int’l|运营商级优化路径|✅ 延迟低  <br>❌ 区域差异大|电信/移动用户优选|

## ⚙️ 二、节点属性标签解析

|   |   |   |
|---|---|---|
|🏷️ 标签|💡 意思|📘 说明|
|🧩 **原生**|使用该国家真实出口IP|可解锁当地Netflix / ChatGPT|
|🔁 **中转**|国内跳板再出国|稳定性取决中转服务器|
|🧠 **BGP动态**|动态最优路由|自动选择延迟最低出口|
|🛰 **IEPL/IPLC**|企业专线|几乎无丢包，超稳定|
|🌍 **Anycast**|全球动态IP分配|自动选最快节点|
|☁️ **AWS / GCP / Azure**|云平台节点|成本低，速度中等|
|⚡ **直连 / 原生出口**|无中转直连国外|速度快但风险高（易封）|

## 🔐 三、协议类型对比

不同协议 = 不同加密方式、伪装方式、抗封锁能力

|   |   |   |   |   |   |
|---|---|---|---|---|---|
|协议|🔤 全称|✨ 特点|👍 优点|⚠️ 缺点|💡 适合人群|
|💨 **Shadowsocks (SS)**|轻量加密代理|速度快、兼容性强|✅ 延迟低|❌ 易被识别|一般用户|
|🔰 **ShadowsocksR (SSR)**|SS 改进版|增强混淆防检测|✅ 稍更抗封锁|❌ 旧协议|老机场|
|🧩 **Vmess (V2Ray)**|V2Ray核心协议|可自定义伪装|✅ 稳定强|❌ 稍复杂|进阶玩家|
|🚀 **Vless (V2Ray)**|无状态轻量协议|性能高|✅ 速度快|❌ 需额外TLS|主流机场|
|🛡 **Trojan**|TLS伪装协议|完全模拟HTTPS|✅ 抗封锁最强|❌ 稍重|大陆用户推荐|
|⚡ **Hysteria / Hysteria2**|UDP加速协议|面向高延迟环境|✅ 延迟极低|❌ 客户端少|游戏、办公|
|🔮 **Tuic**|新一代UDP传输协议|高效低延迟|✅ 极速稳定|❌ 兼容少|新兴高端协议|

## 🧠 五、术语小百科

|   |   |
|---|---|
|术语|含义|
|🧩 **原生IP**：该国家真实分配的IP，不共用，能解锁本地内容。||
|🌀 **中转节点**：通过国内服务器转发，速度取决中转机带宽。||
|🌐 **直连节点**：从本地直接出国，风险高、容易被封。||
|⚙️ **订阅链接**：机场提供的配置自动更新地址。||
|🔁 **测速模式**：可切换为 Ping / TCP / HTTP 三种方式测试延迟。||
|🧱 **被墙**：节点封锁或协议识别导致连接失败。||
|🧭 **延迟**：单位ms，越低越快，通常 <150ms 体验良好。||

## ✨ 建议收藏

✅ **快速判断口诀：**

##### 稳定看 IEPL，便宜选 BGP；

##### 解锁要原生，抗封锁用 Trojan；

##### 游戏低延迟，优先 Hysteria！

---

🧰 **实用贴士：**

- 定期刷新订阅（右上角🔄）保持最新节点；
- 测速时可切换到 HTTP 模式；
- 连接失败多半是线路或协议更新；
- 建议机场同时保留多种协议（SS + Trojan）以防断连。
